Detailed Description

Debug overlay for rendering engine and game information in the screen.

Renders two categories of entries.

Dynamic entries: Represented as DynamicEntry. updated via setDynamicEntry each frame. Entries not updated for a full frame are destroyed in beginFrame. Drawn top-left of the screen. These entries are mostly cached in TTF_Text object (no new texture or surface creation) so updating them every frame is decently fine.

Persistent entries: Normal TTF_Text objects set once via addPersistentEntry, it cannot be updated or changed, stay until clearPersistentEntries is called by the destructor. Drawn top-right of the screen.

Overlay starts hidden, call toggle to make it visible. Only draws when visible, the draw call is no-op if the overlay is not visible.

Warning
Stale dynamic entry cleanup breaks if beginFrame is not called at the start of every frame
Note
setDynamicEntry and addPersistentEntry always add entries and run their logic regardless of visibility, callers do not need to be concerned with visibility when adding lines

Adds a persistent entry that doesn't update and stay until the DebugOverlay is destroyed.

The entries added via this are permanent and cannot be changed during runtime. Entries are drawn top-right of the screen in draw.

Use for static info such as engine version, build type, game title. For frequently updated info use setDynamicEntry.

Parameters
entryText to display
colorDisplay color, defaults to green

◆ beginFrame()

void DebugOverlay::beginFrame ( )

Deletes entries that were not updated last frame and marks all entries as stale for this frame.

Dynamic entries not updated since the last beginFrame call are destroyed. All remaining dynamic entries have their DynamicEntry::updated flag reset to false.

Must be called once per frame before any setDynamicEntry calls.

Warning
Skipping this call causes stale entries to persist even after their caller goes out of scope

const std::string &  entry,

Sets a dynamic entry by tag, creating it if it does not exist.

Drawn top-left of the screen. If an entry with the given tag already exists its text and color are updated in place via TTF_SetTextString. If the tag is new a new TTF_Text object is created.

Entries not updated for a full frame are automatically destroyed in beginFrame. Use for frequently updated info like player position or fps. For static info use addPersistentEntry.

Parameters
tagIdentifier for this entry, only used internally
entryText to display
colorDisplay color, defaults to white
